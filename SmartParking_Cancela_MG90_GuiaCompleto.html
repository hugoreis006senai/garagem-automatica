Visão Geral do Sistema

O Smart Parking IoT monitora 2 vagas de estacionamento com sensores infravermelhos (FC-51). Cada vaga possui um LED verde (livre) e um LED vermelho (ocupado). O ESP8266 publica o estado das vagas via MQTT, controla uma cancela MG90S e o dashboard web exibe tudo em tempo real com botão de manutenção.

🚧 NOVO — Cancela MG90S:
A cancela abre automaticamente quando há vagas disponíveis e fecha quando as duas vagas estiverem ocupadas. O modo manutenção permite controle manual via dashboard, desativando a lógica automática.
Fluxo resumido:
Sensor IR detecta veículo → ESP8266 lê pino → Acende LED correto → Avalia vagas → Move cancela → Publica status via MQTT → Dashboard atualiza em tempo real

Arquitetura em 3 camadas

🔌
Camada de Percepção (Hardware)
2× Sensor IR FC-51, 2× LED Verde, 2× LED Vermelho, resistores, 1× Servo MG90S (cancela), ESP8266, protoboard e jumpers.
📡
Camada de Comunicação (MQTT/Wi-Fi)
ESP8266 conecta ao Wi-Fi e publica nos tópicos de vagas, total livre e status/comando da cancela no broker HiveMQ público.
🖥️
Camada de Aplicação (Dashboard)
Página HTML com JavaScript assina os tópicos MQTT via WebSocket, exibe o mapa de vagas, status da cancela e botão de manutenção em tempo real.

Configuração da Arduino IDE

1
Baixe e instale a Arduino IDE 2.x
Acesse https://www.arduino.cc/en/software e instale normalmente.
2
Adicione suporte ao ESP8266
Em Arquivo → Preferências, cole no campo de URLs adicionais: 
https://arduino.esp8266.com/stable/package_esp8266com_index.json
3
Instale o pacote ESP8266
Vá em Ferramentas → Placa → Gerenciador de Placas. Pesquise "esp8266" e instale o pacote "esp8266 by ESP8266 Community".
4
Selecione a placa correta
Vá em Ferramentas → Placa → esp8266 → NodeMCU 1.0 (ESP-12E Module).
Depois selecione a Porta COM correspondente.
Windows: Se a porta COM não aparecer, instale o driver CH340 ou CP2102 conforme o chip USB do seu ESP8266.

Código Completo do ESP8266 (Firmware)

Altere obrigatoriamente: As linhas WIFI_SSID e WIFI_PASSWORD com o nome e senha da sua rede Wi-Fi (somente 2.4GHz).
FIRMWARE · smart_parking_cancela_MG90.ino
/*
  ============================================================
  SENAI IoT – Grupo 5
  Smart Parking IoT – 2 Vagas + Cancela MG90S
  ESP8266 + Sensores IR FC-51 + LEDs + Servo MG90S via MQTT
  Cancela ABERTA quando há vaga, FECHADA quando tudo ocupado
  Modo manutenção via MQTT para controle manual
  ============================================================
*/

#include <ESP8266WiFi.h>
#include <PubSubClient.h>
#include <Servo.h>

// ── Wi-Fi ────────────────────────────────────────────────
const char* WIFI_SSID     = "SEU_WIFI_AQUI";   // ← ALTERE
const char* WIFI_PASSWORD = "SUA_SENHA_AQUI";  // ← ALTERE

// ── MQTT ─────────────────────────────────────────────────
const char* MQTT_BROKER   = "broker.hivemq.com";
const int   MQTT_PORT     = 1883;
const char* CLIENT_ID     = "senai_grupo5_parking_esp8266";

// ── Tópicos MQTT ─────────────────────────────────────────
const char* TOPIC_VAGA1          = "senai/g93611/grupo5/parking/vaga1";
const char* TOPIC_VAGA2          = "senai/g93611/grupo5/parking/vaga2";
const char* TOPIC_TOTAL          = "senai/g93611/grupo5/parking/total_livre";
const char* TOPIC_CANCELA_STATUS = "senai/g93611/grupo5/parking/cancela/status";
const char* TOPIC_CANCELA_CMD    = "senai/g93611/grupo5/parking/cancela/cmd";

// ── Pinos dos Sensores IR ─────────────────────────────────
#define SENSOR_VAGA1   D3
#define SENSOR_VAGA2   D4

// ── Pinos dos LEDs ────────────────────────────────────────
#define LED_VERDE_V1   D5
#define LED_VERM_V1    D6
#define LED_VERDE_V2   D7
#define LED_VERM_V2    D8

// ── Pino do Servo (Cancela MG90S) ────────────────────────
#define SERVO_CANCELA  D2

// ── Ângulos da Cancela ────────────────────────────────────
#define ANGULO_ABERTA   90  // Cancela levantada
#define ANGULO_FECHADA   0  // Cancela abaixada

const long INTERVAL = 3000;

// ── Variáveis de controle ─────────────────────────────────
bool estadoAnteriorV1   = true;
bool estadoAnteriorV2   = true;
bool cancelaAbertaAtual = true;
bool modoManutencao     = false;
unsigned long lastPublish = 0;

Servo servoCancela;
WiFiClient espClient;
PubSubClient mqtt(espClient);

// ── Mover Cancela ─────────────────────────────────────────
void moverCancela(bool abrir) {
  servoCancela.write(abrir ? ANGULO_ABERTA : ANGULO_FECHADA);
  cancelaAbertaAtual = abrir;
  delay(500);
  mqtt.publish(TOPIC_CANCELA_STATUS,
               abrir ? "ABERTA" : "FECHADA", true);
}

// ── Callback MQTT (comandos de manutenção) ────────────────
void mqttCallback(char* topic, byte* payload, unsigned int length) {
  String msg = "";
  for (unsigned int i = 0; i < length; i++) msg += (char)payload[i];
  msg.toUpperCase();
  msg.trim();

  if (String(topic) == TOPIC_CANCELA_CMD) {
    if (msg == "ABRIR") {
      modoManutencao = true;
      moverCancela(true);
      mqtt.publish(TOPIC_CANCELA_STATUS, "ABERTA (MANUTENCAO)", true);
    }
    else if (msg == "FECHAR") {
      modoManutencao = true;
      moverCancela(false);
      mqtt.publish(TOPIC_CANCELA_STATUS, "FECHADA (MANUTENCAO)", true);
    }
    else if (msg == "AUTO") {
      modoManutencao = false;
      mqtt.publish(TOPIC_CANCELA_STATUS, "MODO AUTOMATICO", true);
    }
  }
}

void connectWiFi() {
  Serial.println(F("\n[WiFi] Iniciando conexão..."));
  WiFi.mode(WIFI_STA);
  WiFi.begin(WIFI_SSID, WIFI_PASSWORD);
  int t = 0;
  while (WiFi.status() != WL_CONNECTED) {
    delay(500); Serial.print('.');
    if (++t >= 30) ESP.restart();
  }
  Serial.println(WiFi.localIP());
}

void connectMQTT() {
  while (!mqtt.connected()) {
    if (mqtt.connect(CLIENT_ID)) {
      mqtt.subscribe(TOPIC_CANCELA_CMD);
      Serial.println(F("[MQTT] OK — inscrito em cancela/cmd"));
    } else { delay(5000); }
  }
}

void atualizarLEDs(int pv, int pr, bool livre) {
  digitalWrite(pv, livre ? HIGH : LOW);
  digitalWrite(pr, livre ? LOW  : HIGH);
}

void publicarStatus(const char* topico, bool livre, int n) {
  String p = "{\\"vaga\\":\\" + String(n) + ",\\"status\\":\\";
  p += livre ? "LIVRE" : "OCUPADO";
  p += ",\\"livre\\":\\" + String(livre ? "true" : "false") + "}";
  mqtt.publish(topico, p.c_str());
}

void setup() {
  Serial.begin(115200);
  pinMode(SENSOR_VAGA1, INPUT);
  pinMode(SENSOR_VAGA2, INPUT);
  pinMode(LED_VERDE_V1, OUTPUT); pinMode(LED_VERM_V1, OUTPUT);
  pinMode(LED_VERDE_V2, OUTPUT); pinMode(LED_VERM_V2, OUTPUT);

  servoCancela.attach(SERVO_CANCELA);
  moverCancela(true);  // Inicia ABERTA

  atualizarLEDs(LED_VERDE_V1, LED_VERM_V1, true);
  atualizarLEDs(LED_VERDE_V2, LED_VERM_V2, true);

  connectWiFi();
  mqtt.setServer(MQTT_BROKER, MQTT_PORT);
  mqtt.setCallback(mqttCallback);
}

void loop() {
  if (WiFi.status() != WL_CONNECTED) connectWiFi();
  if (!mqtt.connected()) connectMQTT();
  mqtt.loop();

  bool v1Livre = (digitalRead(SENSOR_VAGA1) == HIGH);
  bool v2Livre = (digitalRead(SENSOR_VAGA2) == HIGH);

  if (v1Livre != estadoAnteriorV1) {
    atualizarLEDs(LED_VERDE_V1, LED_VERM_V1, v1Livre);
    estadoAnteriorV1 = v1Livre;
  }
  if (v2Livre != estadoAnteriorV2) {
    atualizarLEDs(LED_VERDE_V2, LED_VERM_V2, v2Livre);
    estadoAnteriorV2 = v2Livre;
  }

  // ── Lógica automática da cancela ─────────────────────────
  if (!modoManutencao) {
    bool deveAbrir = (v1Livre || v2Livre);
    if (deveAbrir != cancelaAbertaAtual) moverCancela(deveAbrir);
  }

  unsigned long now = millis();
  if (now - lastPublish >= INTERVAL) {
    lastPublish = now;
    publicarStatus(TOPIC_VAGA1, v1Livre, 1);
    publicarStatus(TOPIC_VAGA2, v2Livre, 2);
    int total = (int)v1Livre + (int)v2Livre;
    mqtt.publish(TOPIC_TOTAL, String(total).c_str());

    String sc = modoManutencao
      ? (cancelaAbertaAtual ? "ABERTA (MANUTENCAO)" : "FECHADA (MANUTENCAO)")
      : (cancelaAbertaAtual ? "ABERTA" : "FECHADA");
    mqtt.publish(TOPIC_CANCELA_STATUS, sc.c_str(), true);
  }
}

Cancela MG90S — Lógica e Manutenção

Lógica Automática

Vaga 1Vaga 2Estado da CancelaAção
LIVRELIVREABERTA (90°)Ambas livres → entrada permitida
OCUPADOLIVREABERTA (90°)Ainda há 1 vaga → entrada permitida
LIVREOCUPADOABERTA (90°)Ainda há 1 vaga → entrada permitida

Modo Manutenção

O modo manutenção desativa a lógica automática e permite abrir ou fechar a cancela manualmente pelo dashboard. Útil para testes, manutenção ou situações de emergência.

🔧
Ativar modo manutenção + Abrir
Clique no botão "🔧 ABRIR CANCELA" no dashboard. O firmware recebe ABRIR, move o servo para 90° e ativa o modo manutenção. O status no dashboard muda para "ABERTA (MANUTENÇÃO)".
🔒
Ativar modo manutenção + Fechar
Clique no botão "🔒 FECHAR CANCELA" no dashboard. O firmware recebe FECHAR, move o servo para 0° e ativa o modo manutenção. O status muda para "FECHADA (MANUTENÇÃO)".
🤖
Voltar ao modo automático
Clique no botão "🤖 MODO AUTO" no dashboard. O firmware recebe AUTO e retoma o controle automático pelos sensores, abrindo ou fechando imediatamente conforme o estado das vagas.
Ajuste dos ângulos: Se o servo do seu projeto funcionar de forma invertida, troque os valores no firmware: #define ANGULO_ABERTA 0 e #define ANGULO_FECHADA 90. Teste girando o braço do servo manualmente antes de fixar.

const TOPIC_V1     = "senai/g93611/grupo5/parking/vaga1";
const TOPIC_V2     = "senai/g93611/grupo5/parking/vaga2";
const TOPIC_TOTAL  = "senai/g93611/grupo5/parking/total_livre";
const TOPIC_CS     = "senai/g93611/grupo5/parking/cancela/status";
const TOPIC_CMD    = "senai/g93611/grupo5/parking/cancela/cmd";
const BROKER_URL   = "ws://broker.hivemq.com:8000/mqtt";

const dot       = document.getElementById('dot');
const connTxt   = document.getElementById('conn-txt');
const cardV1    = document.getElementById('card-v1');
const cardV2    = document.getElementById('card-v2');
const statusV1  = document.getElementById('status-v1');
const statusV2  = document.getElementById('status-v2');
const totalEl   = document.getElementById('total-livre');
const logBox    = document.getElementById('log');
const cancelaPanel = document.getElementById('cancela-panel');
const cancelaBadge = document.getElementById('cancela-badge');
const cancelaIcon  = document.getElementById('cancela-icon');
const modoTag      = document.getElementById('modo-tag');

function log(msg, type='sys') {
  const t = new Date().toLocaleTimeString('pt-BR', {hour12:false});
  const d = document.createElement('div');
  d.className = 'log-entry';
  d.innerHTML = `<span class="log-ts">[${t}]</span><span class="log-${type}">${msg}</span>`;
  logBox.appendChild(d);
  logBox.scrollTop = logBox.scrollHeight;
  while (logBox.children.length > 80) logBox.removeChild(logBox.firstChild);
}

function atualizarVaga(card, statusEl, livre) {
  card.className = 'vaga-card ' + (livre ? 'livre' : 'ocupado');
  statusEl.textContent = livre ? 'LIVRE' : 'OCUPADO';
}

function atualizarCancela(statusStr) {
  const aberta    = statusStr.includes('ABERTA');
  const manutencao = statusStr.includes('MANUTENCAO');
  const modo = manutencao ? 'manutencao' : (aberta ? 'aberta' : 'fechada');
  cancelaPanel.className = 'cancela-panel ' + modo;
  cancelaBadge.className = 'cancela-badge ' + modo;
  cancelaBadge.textContent = statusStr;
  cancelaIcon.textContent  = aberta ? '🔓' : '🔒';
  if (manutencao) {
    modoTag.textContent = '// MODO: MANUTENÇÃO MANUAL';
    modoTag.className = 'modo-tag manual';
  } else if (statusStr === 'MODO AUTOMATICO' || !manutencao) {
    modoTag.textContent = '// MODO: AUTOMÁTICO';
    modoTag.className = 'modo-tag auto';
  }
}

const clientId = 'senai_parking_web_' + Math.random().toString(36).substr(2,6);
const client = mqtt.connect(BROKER_URL, { clientId, clean: true });

client.on('connect', () => {
  dot.className = 'dot ok';
  connTxt.className = 'ok';
  connTxt.textContent = 'CONECTADO';
  log('Conectado ao broker HiveMQ!', 'sys');
  client.subscribe([TOPIC_V1, TOPIC_V2, TOPIC_TOTAL, TOPIC_CS], { qos: 0 });
  log('Aguardando dados do ESP8266...', 'sys');
});

client.on('message', (topic, payload) => {
  const msg = payload.toString();
  log(`← ${topic.split('/').pop()}: ${msg}`, 'sub');

  if (topic === TOPIC_TOTAL) {
    const n = parseInt(msg);
    totalEl.textContent = isNaN(n) ? '–' : n;
    totalEl.style.color = n === 0 ? '#ef4444' : '#22c55e';
  }
  if (topic === TOPIC_CS) {
    atualizarCancela(msg);
  }
  if (topic === TOPIC_V1 || topic === TOPIC_V2) {
    try {
      const data = JSON.parse(msg);
      const livre = data.livre === true;
      if (topic === TOPIC_V1) atualizarVaga(cardV1, statusV1, livre);
      else                     atualizarVaga(cardV2, statusV2, livre);
    } catch(e) {
      const livre = msg.toUpperCase().includes('LIVRE');
      if (topic === TOPIC_V1) atualizarVaga(cardV1, statusV1, livre);
      else                     atualizarVaga(cardV2, statusV2, livre);
    }
  }
});

function cmdCancela(cmd) {
  if (!client.connected) { log('MQTT não conectado!', 'err'); return; }
  client.publish(TOPIC_CMD, cmd);
  log(`→ CMD cancela: ${cmd}`, 'cmd');
}

client.on('error', err => {
  dot.className = 'dot err';
  connTxt.className = 'err';
  connTxt.textContent = 'ERRO';
  log('Erro MQTT: ' + err.message, 'err');
});
client.on('close', () => {
  dot.className = 'dot err';
  connTxt.textContent = 'Reconectando...';
  log('Conexão encerrada.', 'err');
});

Como Testar o Projeto

1
Monte o circuito seguindo a tabela de pinos da Seção 5. Conecte o servo MG90S em D2 com alimentação de 5V (pino VIN).
2
Ajuste o potenciômetro dos FC-51 até o LED do sensor apagar quando houver objeto na frente.
3
Faça upload do firmware — edite SSID/senha e carregue via Arduino IDE. Abra o Monitor Serial (115200 baud) e confirme que o servo se move para 90° ao iniciar.
4
Teste a lógica automática: cubra ambos os sensores → a cancela deve fechar (0°). Remova um objeto → a cancela abre (90°) automaticamente.
5
Teste o modo manutenção: abra o dashboard, clique em "🔒 FECHAR CANCELA" — o servo deve fechar independentemente dos sensores. Clique em "🤖 MODO AUTO" para retornar ao automático.
6
Verifique o status no dashboard: o painel da cancela muda de cor (verde = aberta, vermelho = fechada, laranja = manutenção) em tempo real.

Solução de Problemas

ProblemaCausa ProvávelSolução
ESP8266 não conecta ao Wi-FiSSID ou senha incorretos / rede 5GHzVerifique credenciais. O ESP8266 só suporta redes 2.4GHz.
Servo não se moveAlimentação insuficiente ou pino erradoConfirme que o fio vermelho está em 5V (VIN), não em 3.3V. Verifique conexão do fio laranja em D2.
Servo treme ou vibraFonte de 3.3V insuficiente para o MG90SUse o pino VIN (5V) ou fonte externa. Nunca alimente o MG90S com 3.3V.
Cancela vai para posição erradaÂngulos invertidos para a montagemTroque ANGULO_ABERTA e ANGULO_FECHADA no código (ex: 0 e 90 → 90 e 0).
Botões de manutenção não funcionamMQTT desconectado ou tópico erradoVerifique conexão MQTT no dashboard (indicador verde). Confirme que o ESP8266 está subscrito em cancela/cmd.
Status da cancela não atualizaTópico retained não recebidoRecarregue o dashboard. O status é publicado com retain=true e chega assim que o MQTT conectar.
Sensor com lógica invertidaModelo FC-51 com saída invertidaAltere no código: digitalRead == LOW para "livre".
Dashboard mostra "ERRO"Porta 8000 bloqueada na redeTente outra rede ou use hotspot do celular. A porta 8000 é necessária para WebSocket.
⚙️ Dica de ângulo: Se o servo girar no sentido errado, ajuste os valores de ANGULO_ABERTA e ANGULO_FECHADA no firmware. Valores entre 0° e 180° são suportados pelo MG90S.
